About & Method

KmerLab is a small, local, open-source utility for exploring the k-mer composition of DNA sequence files. This page explains the concepts and the math behind the numbers.

What is a k-mer?

A k-mer is a substring of length k taken from a longer sequence. For the sequence ACGTA with k = 3, the k-mers are ACG, CGT, and GTA — a sliding window of width k moved one base at a time. Counting how often each k-mer appears gives a compact, alignment-free "fingerprint" of a sequence dataset. k-mer profiles are used for genome assembly, error detection, taxonomic classification, contamination screening, and more.

Canonical k-mers

DNA is double-stranded, so a k-mer and its reverse complement usually represent the same physical sequence read from opposite strands. The reverse complement swaps A↔T and C↔G and reverses the string: the reverse complement of AAT is ATT. A canonical k-mer is the lexicographically smaller of a k-mer and its reverse complement. Counting canonical k-mers merges both strand orientations into one count, which is standard practice in tools like KMC and Jellyfish.

GC content

GC content is the fraction of bases that are G or C. KmerLab computes it over A/C/G/T bases only, so ambiguous characters do not distort the ratio.

Jaccard similarity

Jaccard similarity compares two files by the sets of k-mers they contain, ignoring how often each appears:

J(A, B) = |A ∩ B| / |A ∪ B|

It ranges from 0 (no shared k-mers) to 1 (identical k-mer sets). It is a good quick measure of overlap but is sensitive to sequencing depth and errors, which introduce many low-count k-mers.

Cosine similarity

Cosine similarity compares the k-mer frequency vectors — it accounts for how many times each k-mer occurs, not just presence/absence:

cos(A, B) = (A · B) / (‖A‖ · ‖B‖)

Each file is represented as a vector indexed by k-mer, with counts as values. The metric ranges from 0 (orthogonal) to 1 (identical direction / proportional composition). Because it uses counts, it captures compositional similarity even when absolute depths differ.

Frequency Chaos Game Representation (FCGR)

FCGR maps k-mer frequencies onto a 2k × 2k grid where each cell corresponds to one k-mer, placed by recursively subdividing a square with corners A, C, G, T. The result is a texture-like image that reveals compositional patterns at a glance. KmerLab renders it for k ≤ 8.

Limitations

  • Files are processed in memory; very large genomes may exceed available RAM. A streaming mode is planned.
  • The default upload cap is 50 MB in the web UI.
  • Exact counting is used (no probabilistic sketching), so memory scales with the number of distinct k-mers and with k.
  • Only nucleotide sequences are supported; protein k-mers are not (yet).
  • Similarity metrics are only meaningful when both files use the same k and canonical setting.

Privacy & independence

KmerLab makes no network requests. There is no database, no login, no cloud storage, and no API keys. Your sequence data never leaves your computer.
